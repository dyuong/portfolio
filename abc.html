Research
My first impression when interacting with the current ABC website was that it felt messy, and lacked clarity in where it drew the user’s attention towards. The navigation was also confusing, with the primary navigation bar changing between webpages, and there being no clear CTA button to return back to the homepage.

This made the ABC website a perfect choice to redesign.

The first step however was to determine if other user’s found the website to be as arduous to use as I did, as well as to complete a more in-depth heuristic evaluation of the website.
Heuristic Evaluation
Based on Nielsen’s 10 usability heuristics, I conducted a more in-depth analysis on the usability of the ABC website. This allowed me to identify in more detail the pain points users might encounter when interacting with the website, and what I needed to focus on when it came time for the redesign.
Usability Testing
I also completed 5 usability tests, with the participants completing set tasks interacting with the ABC website. Example of tasks were navigating to ABC News and returning to the home page, and listening to the latest episode of a news podcast.

After completing the usability testing, I organised the data using an affinity diagram and prioritisation matrix.

Key Insights
From the research, there were a few main issues that were identified in both the heuristic evaluation and the usability testing:
  • The primary navigation bar changes between webpages which can be confusing for users
  • There is no clear CTA to return to the homepage from the primary navigation
  • There is an overload of information on the homepage
  • Information on the homepage isn’t laid out intuitively
Problem Statement/HMW's
Based off the key insights discovered, we were able to define a problem statement to be answered in our redesign.
How might we ensure that creating an itinerary is as simple and as straightforward as possible?

How might we allow for a collaborative travel planning method which takes into account differences in preferences, values and methods of planning?

Card Sorting
As there were some usability issues with the current website’s navigation, it was important to define the information architecture for the redesigned navigation prior to prototyping.

To do this, I completed an open card session to to first define the contents of the current ABC website navigation, before regrouping and structuring the contents into a new navigation system which I felt was more intuitive.
Sitemap
Based off the card sorting session, I was able to create newly designed sitemap for my redesign of the ABC website.

Testing
After completing the prototype for my redesigned website, it was time to validate my ideas through usability tests. Participants were asked to complete the same tasks that were presented in the initial round of usability testing for the current ABC website.
Overall the feedback was quite positive, with users having no difficulties completing the set tasks. Some of the received feedback is displayed below:
Positive Comments
  • Users found primary navigation easy and intuitive to use

  • Users commented on how simple and clean the website looked

  • Users found layout of information was intuitive, making information easy to find

Constructive Feedback
  • Users preferred the primary and secondary navigation bars to be different colours

  • Users preferred buttons to be filled in as opposed to transparent with a stroke

Taking the feedback into consideration, I continued to iterate and refine my design, eventuating in the final clickable prototype.

Reflections
Aesthetic and Minimalist Design
Keeping in line with Nielsen’s usability heuristics, the new design of the ABC website was kept as clean and simple as possible, with only necessary information being displayed to the user.

This was frequently commented on as a positive in the feedback of the usability tests, and reiterated the fact that keeping designs minimalist allows users to complete their desired tasks with ease, improving their overall satisfaction with the experience.
Principle of Mobile-First Design
In redesigning the website, I started with the desktop device first as I was not familiar with the concept of mobile-first design prior to this project. This led to some issues when trying to make the website responsive across devices, with components not transferring across well from desktop to mobile.

In reflection, leading with the mobile design seems to be best practice in most cases, with it being easier to upscale across larger devices as opposed to downscaling to smaller ones.
